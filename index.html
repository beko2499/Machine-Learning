أنواع Machine Learning

3 أنواع رئيسية: Supervised, Unsupervised, Reinforcement

🎓

فكّر فيها مثل أساليب التعليم

Supervised = مدرس خصوصي (يعطيك السؤال والجواب وتتعلم) | Unsupervised = تعلم ذاتي (تكتشف الأنماط بنفسك) | Reinforcement = تعلم من التجربة (تحاول وتنجح أو تفشل وتتعلم)

📋

1. Supervised Learning (التعلم المُشرف عليه)

📋 البيانات تحتوي على Input + Output (Labels)

🎓 المودل يتعلم العلاقة بين المدخلات والمخرجات من أمثلة "محلولة"

نوعان فرعيان:

  • 📊 Classification (تصنيف) → يتنبأ بفئات منفصلة (مثل: Spam / Not Spam)
  • 📈 Regression (انحدار) → يتنبأ بقيم مستمرة (مثل: سعر = $350,000)

⚠️ مثال الفاكهة: تعطي المودل صور فاكهة مع أسمائها (labeled) → يتعلم التمييز بين 🍎 تفاحة و 🍌 موزة

🔍

2. Unsupervised Learning (التعلم غير المُشرف)

🔍 البيانات بدون تسميات (Unlabeled)

🧩 المودل يكتشف الأنماط والعلاقات المخفية بنفسه

أنواعه:

  • 🎯 Clustering (تجميع) → يجمع البيانات المتشابهة (مثل: تقسيم العملاء)
  • 🔗 Association (ارتباط) → يكتشف العلاقات (مثل: اللي يشتري خبز يشتري حليب)
  • 📉 Dimensionality Reduction → تقليل عدد الخصائص مع حفظ المعلومات

⚠️ مثال العنب: تعطي المودل صور عنب بألوان مختلفة بدون أسماء → يجمّعها حسب اللون بنفسه (أحمر مع أحمر، أخضر مع أخضر)

🎮

3. Reinforcement Learning (تعلم التعزيز)

🤖 Agent (عميل) يتفاعل مع Environment (بيئة)

🎯 يتعلم من Trial and Error (التجربة والخطأ)

العملية:

  • 1️⃣ Agent يلاحظ State (الحالة الحالية)
  • 2️⃣ يتخذ Action (فعل/إجراء)
  • 3️⃣ يحصل على Reward (مكافأة) أو Penalty (عقوبة)
  • 4️⃣ يتحسن مع الوقت!

💡 مثل تعليم طفل ركوب الدراجة: يحاول (action) → يقع (penalty) → يحاول بطريقة مختلفة → ينجح (reward)!

مقدمة Naive Bayes

ليش "Naive"؟ وليش "Bayes"؟

📧

فكّر فيها كذا - مرشح الرسائل المزعجة

تخيل إنك تشتغل في شركة بريد إلكتروني. كل يوم تجيك آلاف الرسائل، بعضها مهم وبعضها spam. كيف تصنّفها؟ تحسب احتمال إن الرسالة spam بناءً على الكلمات اللي فيها! هذا بالضبط Naive Bayes 📊

🤔

ليش اسمها "Naive Bayes"؟

🔤 "Naive" (ساذج):

لأنها تفترض أن كل Feature مستقل عن الباقي (Independent). يعني: اللون الأصفر للفاكهة ما له علاقة بحجمها الكبير — كل خاصية تُحسب لحالها.

📐 "Bayes":

لأنها مبنية على Bayes Theorem (نظرية بايز) — صيغة رياضية تحسب الاحتمالات الشرطية (Conditional Probabilities).

💡 رغم الافتراض "الساذج": الخوارزمية تعمل بشكل ممتاز في الواقع! خصوصاً في تصنيف النصوص وتصفية البريد المزعج.

⚡

كيف يشتغل Naive Bayes؟ (ملخص سريع)

  • 1️⃣ يحسب الاحتمال (Probability) لكل فئة
  • 2️⃣ يحسب الاحتمال الشرطي لكل خاصية مع كل فئة
  • 3️⃣ يختار الفئة ذات أعلى احتمال (Maximum Probability)

📊 نوعها: Supervised Learning → Classification

🎯 المدخل: Features (خصائص) | المخرج: Class (فئة)

نظرية Bayes

الصيغة الأساسية لكل حسابات Naive Bayes

🔑

فكّر فيها كذا - تحديث المعرفة

نظرية بايز = "كيف تُحدّث رأيك بعد ما تشوف دليل جديد". مثال: تعتقد إن 40% من الرسائل spam (هذا Prior). بعدين تشوف كلمة "مجاني" في الرسالة (هذا Evidence). بايز يساعدك تحسب الاحتمال الجديد (Posterior)!

📐

صيغة Bayes Theorem

P(A|B) = P(B|A) × P(A) / P(B)

🎯

P(A|B) = Posterior

الاحتمال البَعدي

الاحتمال المُحدّث بعد مشاهدة الدليل. هذا ما نريد حسابه!

📊

P(B|A) = Likelihood

الإمكانية

احتمال مشاهدة الدليل B إذا كانت الفرضية A صحيحة

📋

P(A) = Prior

الاحتمال المسبق

ما نعرفه مسبقاً قبل أي دليل جديد

🔍

P(B) = Evidence

الدليل

الاحتمال الكلي للدليل (normalizing constant)

⚠️ مهم للاختبار: P(A|B) تُقرأ "Probability of A given B" = احتمال A بشرط حدوث B. الخط العمودي | يعني "given" (بشرط).

⚡

كيف يستخدم Naive Bayes نظرية بايز؟

إذا عندنا حدث E وعوامل اختبار x₁, x₂, x₃...

  • 1️⃣ نحسب P(x₁|E), P(x₂|E), P(x₃|E)...
  • 2️⃣ نختار العامل x ذو أعلى قيمة احتمال

📌 بالعربي: Naive Bayes يحسب احتمال كل فئة ممكنة، ثم يقول: "الفئة اللي احتمالها الأعلى هي الجواب!"

🎯 النتيجة: P(No|x) = 0.0206 > P(Yes|x) = 0.0053 → القرار: ❌ No — لا نلعب تنس هذا اليوم!

مقدمة SVM

فهم الفكرة الأساسية لـ Support Vector Machine

🎯 تخيل هذا السيناريو...

عندك مجموعة من الدوائر السوداء ● ومجموعة من المربعات الزرقاء ■ على ورقة رسم بياني. مطلوب منك ترسم خط يفصل بينهم. وش تسوي؟

ترسم خط مستقيم بينهم — بحيث كل الدوائر على جانب وكل المربعات على الجانب الآخر! 📏

هذا بالضبط اللي يسويه SVM! بس يسوّيه بأذكى طريقة — يختار الخط اللي يعطي أكبر مسافة فاصلة (Margin) بين الفئتين.

📖 التعريف الرسمي

Given labeled training data (supervised learning), the SVM algorithm outputs an optimal hyperplane which categorizes new examples. In two-dimensional space, this hyperplane is a line dividing a plane into two parts where each class lies on either side.

📌 بالعربي: SVM يأخذ بيانات تدريب مُعنونة ويخرج مستوى فائق أمثل (خط في 2D) يقسم البيانات إلى فئتين.

📏 Hyperplane

المستوى الفائق — في 2D خط مستقيم، في 3D سطح مستوي. يفصل بين الفئات.

📌 Support Vectors

نقاط البيانات الأقرب للـ Hyperplane. هي اللي تحدد موقعه واتجاهه.

📐 Margin

المسافة بين الـ Hyperplane وأقرب نقاط من كل فئة. SVM يكبّر هذه المسافة!

🏷️ Supervised

SVM يحتاج بيانات مُعنونة (labeled) = نعرف الفئة مسبقاً عشان يتعلم.

📏 Hyperplane والهامش

خط الفصل وأهمية Maximum Margin

🎯 الفكرة ببساطة

عندنا مجموعتين من البيانات ونبي نفصلهم بخط. بس مو أي خط — نبي أفضل خط ممكن! الخط اللي يعطينا أكبر مسافة (Margin) من الفئتين.

✅ Good Margin

  • الـ Hyperplane متساوي البعد من الفئتين
  • المسافة أكبر ما يمكن (Equidistant)
  • تصنيف أدق للنقاط الجديدة
  • أقل عرضة للأخطاء

❌ Bad Margin

  • الـ Hyperplane قريب جداً من فئة واحدة
  • تحيز (Bias) واضح
  • أخطاء في تصنيف النقاط الجديدة
  • دقة (Accuracy) منخفضة

🔍 Outliers والقيم الشاذة

لما يكون عندنا Outliers (قيم شاذة خارج النمط) — هذه تأثر على موقع الـ Hyperplane! عشان كذا SVM يوفر 3 معاملات للتحكم:

1️⃣ Regularization (C) — التسامح مع الأخطاء
2️⃣ Kernel — نوع التحويل (Linear أو Polynomial)
3️⃣ Gamma — مدى تأثير النقاط

🔄 Kernel والتحويل

كيف نتعامل مع البيانات غير القابلة للفصل الخطي

💡 المشكلة: بيانات متداخلة!

أحياناً البيانات ما تنفصل بخط مستقيم. مثلاً: مجموعة من النقاط في دائرة ومجموعة حولها — ما ينفع نرسم خط مستقيم يفصلهم!

الحل؟ Kernel Trick! نحول البيانات إلى بُعد أعلى حيث يمكن فصلها.

📐 أنواع الـ Kernel

📏 Linear Kernel

يحسب Inner Product بين المتجه الجديد وكل Support Vectors. مناسب لما البيانات قابلة للفصل بخط مستقيم.

K(x,y) = x · y

🔀 Polynomial Kernel

يحسب خط الفصل في بُعد أعلى (Higher Dimension). يستخدم Kernel Trick للفصل غير الخطي.

K(x,y) = (x · y + c)^d

🪄 ما هي Kernel Trick؟

الـ Kernel Method هي أداة تعمل على تحويل المساحات ذات الأبعاد القليلة إلى مساحات ذات أبعاد أكثر — تحويل المستويات ذات البعدين إلى مستويات ذات ثلاثة أو أربعة أبعاد.

أي أنها تحول المسألة التي لا يمكن فصلها إلى مسألة قابلة للفصل. تفيد هذه الطريقة في الغالب في مسائل التصنيف غير الخطية.

ببساطة: بيانات في 2D ما تنفصل بخط → الكيرنل يحسبها كأنها في 3D → هناك يمكن فصلها بسطح!

⚖️ Regularization (C)

التسامح مع الأخطاء — التنعيم

📖 التعريف

The Regularization parameter tells the SVM optimization how much you want to avoid misclassifying each training example — i.e., the amount of tolerance of outliers when drawing the line.

📌 بالعربي: معامل C يحدد مقدار التسامح مع نقاط التدريب اللي تقع على الجانب الخطأ من الخط.

📈 C عالي (High C)

  • تسامح أقل مع الأخطاء
  • الـ Hyperplane يحاول يفصل كل نقطة صح
  • هامش ضيّق
  • ⚠️ خطر Overfitting

📉 C منخفض (Low C)

  • تسامح أكثر مع الأخطاء
  • هامش أعرض
  • تعميم أفضل على بيانات جديدة
  • ✅ أقل عرضة لـ Overfitting

💡 في Scikit-learn: المعامل يُسمى C (حرف كبير). القيمة الافتراضية = 1.0. كلما زدنا C = الموديل يصير أكثر حساسية لبيانات التدريب.

📡 Gamma

مدى تأثير نقاط التدريب

📖 التعريف

The gamma parameter defines how far the influence of a single training example reaches, with low values meaning "far" and high values meaning "close".

📌 بالعربي: Gamma يحدد كم نقطة تدريب واحدة تأثر على حدود القرار. هل نأخذ النقاط البعيدة في الاعتبار ولا بس القريبة؟

📉 Low Gamma

  • النقاط البعيدة تُؤخذ في الاعتبار
  • منحنى قرار أكثر سلاسة (Smooth)
  • تعميم أفضل على بيانات جديدة ✅
  • أقل عرضة لـ Overfitting

📈 High Gamma

  • فقط النقاط القريبة تُعتبر
  • منحنى قرار معقد ومتعرج
  • ⚠️ خطر Overfitting
  • يتأثر بكل نقطة قريبة

💡 نصيحة للحفظ

Low Gamma = Far (بعيد) → النقاط البعيدة تؤثر → منحنى سلس
High Gamma = Close (قريب) → فقط القريبة → منحنى معقد

تذكر: الأسماء عكسية! Low=Far, High=Close 🔄

🧪 أمثلة عملية

تطبيقات SVM في الحياة الواقعية

🏥 مثال Obesity (البدانة)

لدينا بيانات عن أشخاص: الطول (Height) والوزن (Weight). نبي نصنّف: هل الشخص Obese أو Not Obese؟

❌ خط متحيز

فيه تحيز وعدم دقة — الخط قريب من فئة ويخطئ في التصنيف.

✅ خط أفضل

أفضل ويصنف النقاط صح — يفصل بشكل متساوي بين Obese و Not Obese.

📊 تقسيم البيانات (Train/Test Split)

📚 Training Set — 70%

البيانات اللي يتعلم منها الموديل. يرسم الـ Hyperplane بناءً عليها.

🧪 Testing Set — 30%

البيانات اللي نختبر بيها الموديل. ما يشوفها أثناء التدريب!

💊 تطبيق: تحديد تأثير فعالية الدواء

SVM يمكن استخدامه لتصنيف المرضى بناءً على استجابتهم للدواء. يرفع البيانات لـ Higher Dimensional Space باستخدام Kernel Trick عشان يلاقي الفصل الأمثل بين المستجيبين وغير المستجيبين.

# إنشاء الموديل
clf = svm.SVC(kernel='linear', C=1.0, gamma='scale')

# التدريب
clf.fit(X_train, y_train)

# التنبؤ
y_pred = clf.predict(X_test)

# الدقة
from sklearn.metrics import accuracy_score
print(accuracy_score(y_test, y_pred))

فهم الفكرة الأساسية لشجرة القرار

🎯 تخيل هذا السيناريو...

صديقك يسألك: "هل ألعب كرة اليوم؟"

أنت تسأله سلسلة أسئلة: هل الجو مشمس؟ → هل في رياح؟ → هل الرطوبة عالية؟

بناءً على إجاباته، توصل للقرار! هذا بالضبط اللي تسويه Decision Tree — سلسلة من الأسئلة تقود لقرار نهائي. 🌳

📖 التعريف الرسمي

A Decision Tree is a non-parametric supervised learning method used for both classification and regression. The goal is to create a model that predicts the value of a target variable by learning simple decision rules inferred from the data features.

📌 بالعربي: Decision Tree هي خوارزمية تعلم مُشرف تبني نموذج يتنبأ بالقيم عن طريق تعلم قواعد قرار بسيطة من خصائص البيانات.

📊 كيف تعمل؟

Decision Trees work on dividing the data into subsections using separating lines. This is continued until the classes perfectly describe the data or a certain criteria is met.

📌 تقسم البيانات إلى أقسام فرعية باستخدام خطوط فاصلة. تستمر حتى الفئات تصف البيانات بشكل مثالي.

✅ مميزات Decision Trees

  • سهلة الفهم — ليست Black Box!
  • تعمل لـ Classification و Regression
  • Non-parametric — لا تحتاج افتراضات
  • تتعامل مع بيانات فئوية وعددية

⚠️ العيوب

  • Overfitting — الشجرة تصير عميقة جداً
  • حساسة للتغييرات الصغيرة
  • قد تكون غير دقيقة بمفردها
  • تحتاج Pruning أو Ensemble Methods

🏗️ بنية الشجرة

المكونات الأساسية لشجرة القرار

🌳 Root Node

العقدة الجذرية — أعلى عقدة في الشجرة. السؤال الأول. يتم اختيارها بناءً على أعلى Information Gain.

🔀 Branch

الفرع — الخطوط اللي تربط العقد. كل فرع يمثل إجابة محتملة للسؤال (مثلاً: Yes/No, Sunny/Rainy).

🍃 Leaf Node

العقدة الورقية — النقطة النهائية. تمثل القرار النهائي (التصنيف). لا تُقسم أكثر.

⚡ Impurity

عدم النقاء — وجود آثار من فئة في تقسيم فئة أخرى. الهدف: تقليلها واختيار التقسيم الأنقى.

💡 كيف تُقرأ الشجرة؟

تبدأ من Root Node (أعلى). لكل سؤال، اتبع الفرع المناسب لإجابتك. استمر حتى تصل لـ Leaf Node — هذا هو القرار النهائي!

ندخل البيانات، وبناءً على مجموعة أسئلة على الإجابات يتم التوقع. الخوارزمية تختار X & Y والعلاقة بينهم بناءً على إجابات الأسئلة.

🎲 Entropy

درجة العشوائية وعدم النقاء

📖 التعريف

Entropy is the degree of randomness of elements or in other words it is a measure of impurity. Mathematically, it can be calculated with the help of probability of the items.

📌 بالعربي: Entropy = مقياس لدرجة العشوائية (عدم النقاء). يُحسب باستخدام احتمالات العناصر. دائماً يُطبق على العمود الأخير (Target Variable).

E = -Σ p(i) × log₂(p(i))

✅ Entropy = 0

نقي 100% — كل العناصر من نفس الفئة. أفضل حالة! لا حاجة لتقسيم أكثر.

❌ Entropy = 1

أقصى عشوائية — 50/50 بين فئتين. أسوأ حالة! التصنيف مثل رمي عملة.

📝 مثال عملي: رمي نرد

إذا رمينا نرد وحصلنا على: 1, 1, 2, 3

P(1) = 2/4 = 0.5
P(2) = 1/4 = 0.25
P(3) = 1/4 = 0.25

E = -(0.5 × log(0.5)) - (0.25 × log(0.25)) - (0.25 × log(0.25))
E ≈ 0.45

💡 تذكر: Entropy كلما قيمته قليلة = أفضل! (عكس Information Gain)

📊 Information Gain

كسب المعلومات — كيف نختار أفضل سؤال

📖 التعريف

Information Gain measures how much "information" a feature gives us about the class. It is the reduction in entropy after splitting on a feature.

📌 بالعربي: Information Gain يقيس كم معلومات نكسب من تقسيم معين. كلما زاد IG = التقسيم أفضل!

IG = Entropy(parent) - [Weighted Average × Entropy(children)]

📉 Entropy

  • مقياس العشوائية
  • يتراوح بين 0 و 1
  • كلما قلّ = أفضل ✅
  • يُطبق على العمود الأخير

📈 Information Gain

  • مقياس جودة التقسيم
  • كلما زاد = أفضل ✅
  • يُحسب لكل Feature
  • أعلى IG = Root Node

🔑 كيف نستخدمه لبناء الشجرة؟

1️⃣ احسب Entropy للعمود الأخير (Target)

2️⃣ لكل Feature: احسب Entropy بعد التقسيم ← احسب IG

3️⃣ الـ Feature بأعلى IG = Root Node

4️⃣ كرر العملية لكل فرع حتى تصل لـ Leaf Nodes نقية

📈 مقدمة Linear Regression

فهم الفكرة الأساسية للانحدار الخطي

🎯 تخيل هذا السيناريو...

أنت وكيل عقارات. عميل يسألك: "كم سعر منزل مساحته 1800 قدم²؟"

عندك بيانات مبيعات سابقة — مساحة المنزل وسعره. تلاحظ إن كلما زادت المساحة → زاد السعر. هذا بالضبط اللي يسويه Linear Regression — يرسم أفضل خط مستقيم عبر البيانات ليتنبأ بالقيم الجديدة! 📈

📖 التعريف الرسمي

It's a method to predict a target variable by fitting the best linear relationship with the least error between the dependent and independent variable.

📌 بالعربي: Linear Regression = طريقة للتنبؤ بمتغير مستهدف عن طريق إيجاد أفضل علاقة خطية بأقل خطأ بين المتغير التابع والمستقل.

📊 Supervised Learning

تعلم مُشرف — عندنا "right answers" أو "labeled data" مسبقاً.

📈 Continuous Output

يتنبأ بقيم مستمرة (سعر، درجة، إيرادات) — وليس فئات Yes/No.

🔗 Independent Variable (X)

المتغير المستقل — المدخل. مثل: مساحة المنزل، ساعات المذاكرة.

🎯 Dependent Variable (Y)

المتغير التابع — المخرج المطلوب توقعه. مثل: السعر، الدرجة.

🔀 أنواع الانحدار الخطي

Simple vs Multiple

📊 Simple Linear Regression

  • متغير مستقل واحد (X)
  • متغير تابع واحد (Y)
  • h(x) = θ₀ + θ₁x
  • مثال: ساعات مذاكرة → درجة

📊 Multiple Linear Regression

  • عدة متغيرات مستقلة (X₁, X₂...)
  • متغير تابع واحد (Y)
  • h(x) = θ₀ + θ₁x₁ + θ₂x₂ + ...
  • مثال: مساحة + غرف + موقع → سعر

📊 أنواع العلاقات

↗️ Positive Linear

علاقة طردية: X يزيد → Y يزيد. مثل: ساعات مذاكرة ↑ درجة ↑

↘️ Negative Linear

علاقة عكسية: X يزيد → Y يقل. مثل: سرعة ↑ وقت وصول ↓

➡️ No Relationship

لا توجد علاقة بين X و Y. الخط أفقي، θ₁ = 0.

〰️ NOT Linear

العلاقة موجودة لكن ليست خطية. نحتاج نموذج آخر (Polynomial).

🧮 تمثيل النموذج

Training Set, Hypothesis, والمتغيرات

📋 Training Set — مجموعة التدريب

مثال: بيانات أسعار المنازل في Portland, OR

Size (feet²) → X | Price ($1000s) → Y
2104 → 460 | 1416 → 232 | 1534 → 315 | 852 → 178

m = عدد أمثلة التدريب (عدد الصفوف)
x's = "input" variable / features
y's = "output" / "target" variable
(x⁽ⁱ⁾, y⁽ⁱ⁾) = المثال التدريبي رقم i (صف واحد)

🔄 كيف يعمل النموذج؟

The job of a learning algorithm is to output a function, usually denoted h (hypothesis). The job of h is taking the value of x and tries to output the estimated value of y.

Training Set → Learning Algorithm → h
x → h → ŷ (predicted y)

📌 بالعربي: ندخل البيانات → الخوارزمية تتعلم → تنتج دالة h → ندخل x جديد → تعطينا التنبؤ ŷ

📐 دالة الفرضية (Hypothesis)

hθ(x) = θ₀ + θ₁x

💡 الفكرة: اختر θ₀ و θ₁ بحيث hθ(x) تكون قريبة من y لأمثلة التدريب (x, y).

📐 Slope و Intercept

الميل ونقطة التقاطع — فهم المعادلة

📍 θ₀ = Y-intercept

نقطة التقاطع مع محور Y — القيمة المتوقعة لـ Y عندما X = 0. تغييره = تحريك الخط لأعلى/أسفل بشكل موازي (الميل ثابت).

📈 θ₁ = Slope (الميل)

معدل التغير = ΔY / ΔX — كم تتغير Y لكل وحدة X. تغييره = دوران الخط (تغيير الزاوية).

📊 حساب Slope و Intercept

Slope (a) = Σ(xᵢ - x̄)(yᵢ - ȳ) / Σ(xᵢ - x̄)²

Intercept (b) = ȳ - a × x̄

حيث x̄ و ȳ = المتوسطات

🔍 تأثير تغيير θ₀ و θ₁

تغيير θ₀ فقط

الميل ثابت → الخط يتحرك لأعلى أو لأسفل بشكل موازي (Slope unchanged, Intercept changed).

تغيير θ₁ فقط

الخط يدور → يتغير ميله/زاويته (Slope changed, Intercept changed).

θ₁ > 0 (موجب)

علاقة طردية ↗️ كلما زاد X → زاد Y.

θ₁ < 0 (سالب)

علاقة عكسية ↘️ كلما زاد X → قل Y.

📉 Cost Function

دالة التكلفة — كيف نختار أفضل خط

🤔 المشكلة: كيف نختار θ₀ و θ₁؟

The cost function lets us figure out how to fit the best possible straight line to our data. How to choose θᵢ's?

📌 عندنا scatter plot — كيف نحدد أي خط "يلائم" البيانات أفضل؟ الجواب: Cost Function!

📐 Least Squares — المربعات الصغرى

'Best Fit' means the difference between actual Y values and predicted Y values is a minimum. So we square the errors!

SSE = Σ(Yᵢ - h(xᵢ))²
Sum of Squared Errors — مجموع مربعات الأخطاء

📌 ε (إبسلون) = الخطأ = الفرق بين القيمة الفعلية والمتوقعة. الهدف: تقليل مجموع مربعات الأخطاء!

❓ ليش نربّع الأخطاء؟

1️⃣ إلغاء السالب

بدون تربيع: +3 و -3 يلغيان بعض = 0. بالتربيع: 9 + 9 = 18 ✅

2️⃣ عقاب الأخطاء الكبيرة

خطأ 2 → 4 | خطأ 10 → 100! الأخطاء الكبيرة تُعاقب أكثر بكثير.

3️⃣ قابلية الاشتقاق

التربيع يجعل الدالة سلسة وقابلة للاشتقاق — مهم لخوارزميات التحسين.

y = b₀ + b₁ × x₁

b₀ = Constant

المعامل (Slope) — مقدار تأثير x على y.

📉 Cost Function

قياس أداء النموذج

📖 التعريف

Cost Function is a function that measures the performance of a Machine Learning model for given data. It calculates the error between predicted values and expected values and presents it as a single real number.

📌 بالعربي: دالة تقيس أداء النموذج بحساب الخطأ بين التنبؤات والقيم الفعلية وتقدمه كرقم واحد. أقل = أفضل!

📊 أنواع دوال التكلفة

📊 SSE

Sum of Squared Errors
SSE = Σ(yᵢ - ŷᵢ)²
مجموع مربعات الأخطاء. يزيد مع زيادة البيانات.

📊 MSE

Mean Squared Error
MSE = (1/m) × Σ(yᵢ - ŷᵢ)²
متوسط مربعات الأخطاء. أفضل للمقارنة.

⬇️ Gradient Descent

النزول التدرجي — كيف يتعلم النموذج

🏔️ تخيل هذا...

أنت واقف على قمة تل في ضباب كثيف. تريد النزول لأدنى نقطة لكن لا ترى!

ماذا تفعل؟ تحس الأرض تحت قدميك → تمشي باتجاه الانحدار → خطوة خطوة → حتى تصل لأسفل!

هذا بالضبط Gradient Descent! 🎯 الكرة = المعاملات θ | أسفل التل = أقل تكلفة.

📖 التعريف

It iteratively moves towards a set of values that minimise the loss function.

📌 خوارزمية تتحرك تكرارياً نحو القيم اللي تقلل دالة الخسارة.

📋 خطوات Gradient Descent

1️⃣ Initialize

ابدأ بقيم عشوائية لـ θ₀ و θ₁

2️⃣ Calculate Cost

احسب J(θ) = (1/2m) × Σ(h(x)-y)²

3️⃣ Calculate Gradient

احسب المشتقة ∂J/∂θ (اتجاه الزيادة)

4️⃣ Update
